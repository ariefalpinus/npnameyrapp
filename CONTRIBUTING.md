## How to Contribute

### 1. Fork the Repository

Click the Fork button in GitHub and clone your fork:

```bash
git clone https://github.com/<your-username>/npnameyrapp.git


### 2. Create a Feature Branch

Please avoid working in the `main` branch.

```bash
git checkout -b feature/my-new-feature


### 3. Make Your Changes

Follow the existing coding style.
Keep functions modular and documented.
Add comments when code decisions may not be obvious.
If you add a new dependency, update the documentation.

### 4. Test Your Changes

Before submitting:

Ensure the Shiny app runs without errors.
Test on multiple platforms if possible (Windows / Linux / RStudio / Conda).
Validate UI inputs and outputs, particularly reactive components.

### 5. Commit Your Work

Use clear, descriptive commit messages:

```bash
git commit -m "Add kernel density export option"


### 6. Push and Submit a Pull Request

```bash
git push origin feature/my-new-feature


Then open a Pull Request (PR) on GitHub:

Describe what you changed and why.
Reference related issues if applicable.
Include screenshots for UI changes.

---

##  Contribution Guidelines

### Code Style

Use **tidyverse style** where appropriate.
Indentation: **2 spaces**.
Avoid overly long lines (> 120 characters).
Use informative variable names.

### Documentation

Update `README.md` if your change affects user workflow.
Add comments for any new functions.
For major features, include usage examples.

### Issues & Feature Requests

Before opening a new issue, search existing ones.
Provide reproducible examples when reporting bugs.
Use labels if possible (bug, enhancement, question).
