# Frequently Asked Questions (FAQ)

This document answers common questions about **npnameyrapp v1.0**.  
If your question is not listed here, please check the [Troubleshooting](#) section in the README or contact the project team.

---

## Installation & Setup

**Q: What version of R is required?**  
A: R 4.3.3 or later is recommended. The app may work on earlier versions, but compatibility is not guaranteed.

**Q: Do I need RStudio to run the app?**  
A: RStudio is recommended for ease of use, but you can run the app from any R environment.

**Q: Where should I place `CT_TABLE.txt`?**  
A: Place it in the folder:  

<working_directory>/data/ctTable/

---

## Running the Application

**Q: How do I launch App1 and App2?**  
A: The procedure is the same:  
1. Confirm your working directory with `setwd()`.  
2. Run `source("KEY_APP/launch-app.R")` or `source("KEY_APP/launch-app2.R")`.  
3. Start the app with `shinyApp(ui, server)`.

**Q: What outputs are generated by App1?**  
A: App1 produces:  
- Record Table  
- Camera Operation Matrix  
- Survey Report  
- `App1_output.RData`

**Q: What outputs are generated by App2?**  
A: App2 produces:  
- GBIF validation table  
- RAI table  
- Kernel Density Plot (PNG, 600 dpi)  
- Static Occupancy Table

## Dependencies

**Q: Which R packages are required?**  
A: The app depends on:  
- `shiny`  
- `camtrapR`  
- `overlap`  
- `RPresence`  
- plus other packages listed in the README and CITATION.cff.

**Q: Do I need to install `exiftool.exe`?**  
A: Yes. Place `exiftool.exe` in a folder (e.g., `C:/Windows`) and set the path in the app.

## Troubleshooting

**Q: The app doesn’t start — what should I check first?**  
A: Confirm:  
- Your working directory is set correctly.  
- Required packages are installed.  
- `CT_TABLE.txt` is in the correct folder.  

**Q: Where can I ask for help?**

`npshinyapp@googlegroups.com`  
All troubleshooting discussions are shared there.
